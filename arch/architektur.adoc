== Projektübersicht

[.lead]
Die BeagleBone Black Embedded SW ist eine mehrschichtige Systemarchitektur
für den *TI Sitara AM335x (ARMv7 Cortex-A8, 1GHz, 512MB RAM)*.
Sie verbindet Hardware-nahe C und Rust Bibliotheken über einen
HAL-Wrapper mit einer Go REST API und mehreren Client-Tools.

=== Projektsteckbrief

[cols="2,3", options="header", width="80%"]
|===
| Eigenschaft | Wert

| Plattform | BeagleBone Black (ARMv7 Cortex-A8)
| Prozessor | TI Sitara AM335x, 1 GHz
| Arbeitsspeicher | 512 MB DDR3
| Flash | 4 GB eMMC
| Betriebssystem | Debian Linux, Kernel 5.x (ARMv7 HF)
| Primärsprachen | Go, C, Rust
| Container Runtime | Podman (rootless)
| CI/CD | Drone CI + Gitea
| Architektur-Schichten | 5 (Client → API → HAL → Libraries → Hardware)
|===

=== Systemziele

[NOTE]
====
Das System verfolgt folgende primäre Ziele:

* *Hardware-Abstraktion*: Einheitlicher Zugriff auf GPIO, I2C, UART, SPI
  unabhängig vom Backend (C oder Rust)
* *Sicherheit*: Rust als speichersicheres Backend, C als leistungsoptimiertes
  Backend
* *Beobachtbarkeit*: REST API + SSE Streams für Echtzeit-Monitoring
* *Testbarkeit*: Mock Driver für Unit Tests ohne physische Hardware
* *Portabilität*: Cross-Compilation für ARMv7, amd64, darwin, windows
====

==== HAL Interface

Der HAL (Hardware Abstraction Layer) definiert ein einheitliches
Go Interface für alle Hardware-Operationen:

[source,go]
.HAL Interface Definition
----
type HardwareDriver interface {
    Name()    string
    Backend() Backend

    BME280Read()                          (*BME280Data, error)
    GPIOExport(pin uint32)                error
    GPIOSetDirection(pin uint32, out bool) error
    GPIORead(pin uint32)                  (*GPIOData, error)
    GPIOWrite(pin uint32, value int)      error
    UARTOpen(port string, baud uint32)    error
    UARTWrite(data []byte)                (int, error)
    UARTRead(timeoutMs int)               (*UARTData, error)
    UARTClose()
    SPITransfer(dev string, speed uint32,
                tx []byte)                (*SPIData, error)
    Init()  error
    Close()
}
----

==== Backend Auswahl

[source,go]
.Factory Pattern — Backend Auswahl
----
func NewDriver(cfg *Config) (HardwareDriver, error) {
    switch cfg.Backend {
    case BackendC:
        return cdriver.New(cfg)    // C via CGO
    case BackendRust:
        return rustdriver.New(cfg) // Rust via FFI
    case BackendAuto:
        return newAutoDriver(cfg)  // C → Rust Fallback
    }
}
----

==== C Library (libhardware.so)

[cols="2,2,3", options="header"]
|===
| Modul | Schnittstelle | Beschreibung

| `bme280.c` | I2C ioctl | BME280 Sensor, Bosch Kompensationsalgorithmus
| `gpio.c` | sysfs | GPIO Export, Direction, Read/Write
| `uart.c` | termios | Serial Port, Baudrate, Parität
| `spi.c` | spidev ioctl | SPI Transfer, Mode 0-3, bis 48 MHz
|===

[source,c]
.BME280 Read — C Implementierung
----
int bme280_read(bme280_dev_t *dev, bme280_data_t *out) {
    uint8_t raw[8];
    if (i2c_read_reg(dev->fd, 0xF7, raw, 8) < 0)
        return -1;

    int32_t raw_temp  = ((int32_t)raw[3]<<12)
                      | ((int32_t)raw[4]<<4)
                      | (raw[5]>>4);
    out->temperature = comp_temp(dev, raw_temp);
    out->pressure    = comp_press(dev, raw_press);
    out->humidity    = comp_hum(dev, raw_hum);
    out->altitude    = calc_altitude(out->pressure);
    return 0;
}
----

==== Rust Library (libhardware_rs.so)

[cols="2,2,3", options="header"]
|===
| Modul | Crate | Beschreibung

| `bme280.rs` | linux-embedded-hal | BME280, memory-safe I2C
| `gpio.rs` | std::fs | sysfs GPIO, ownership-basiert
| `uart.rs` | serialport | Serial Port, async-ready
| `spi.rs` | spidev | SPI Transfer, zero-copy
|===

[source,rust]
.BME280 FFI Export — Rust
----
#[no_mangle]
pub unsafe extern "C" fn rs_bme280_read(
    i2c_path: *const c_char,
    addr: u8,
) -> RsBme280Data {
    let i2c    = I2cdev::new(path)?;
    let mut s  = BME280::new(i2c, addr, &mut Delay)?;
    s.init(&mut Delay)?;
    let m = s.measure(&mut Delay)?;
    RsBme280Data {
        temperature: m.temperature as f64,
        humidity:    m.humidity as f64,
        pressure:    m.pressure as f64 / 100.0,
        error:       0,
    }
}
----

==== BME280 Verkabelung

[source]
.I2C Verbindung — P9 Header
----
BeagleBone Black        BME280
──────────────────────────────
P9_19 (I2C2_SCL)  ───►  SCL
P9_20 (I2C2_SDA)  ───►  SDA
P9_3  (3.3V)      ───►  VCC
P9_1  (GND)       ───►  GND
                  ───►  SDO → GND  (Adresse 0x76)
----

[WARNING]
====
Der BeagleBone Black verfügt über *512 MB RAM*.
Schwere Laufzeitumgebungen (JVM, Python-Interpreter als Daemon) sind
zu vermeiden. Go und Rust erzeugen statische Binaries mit minimalem
Speicherbedarf.
====

== Entscheidungsprotokoll

=== ADR-001: Go als API-Sprache

[cols="2,4"]
|===
| Datum | 2026-01-15
| Status | Akzeptiert
| Entscheider | Embedded SW Team
|===

*Kontext:*
Wahl der Sprache für den REST API Server auf dem BeagleBone Black.

*Entscheidung:*
Go wurde gewählt.

*Begründung:*
- Statische Binaries ohne Runtime-Abhängigkeiten
- Geringe Speichernutzung (~5 MB für den Server)
- Exzellente CGO-Integration für C-Bibliotheken
- Eingebaute HTTP/SSE Unterstützung

*Alternativen:*
Python (zu viel RAM), Node.js (zu groß), Rust (komplexer für REST)

=== ADR-002: C + Rust parallel als Hardware-Libraries

[cols="2,4"]
|===
| Datum | 2026-01-20
| Status | Akzeptiert
|===

*Kontext:*
Wahl der Sprache(n) für direkten Hardware-Zugriff.

*Entscheidung:*
Beide Sprachen parallel mit automatischem Fallback.

*Begründung:*
- *C*: Maximale Performance, direkter sysfs/ioctl Zugriff, bewährt
- *Rust*: Memory Safety, kein undefined behavior, moderne Crates
- *HAL Wrapper*: Abstrahiert die Wahl — API-Code ist identisch

=== ADR-003: Podman statt Docker

[cols="2,4"]
|===
| Datum | 2026-02-01
| Status | Akzeptiert
|===

*Entscheidung:*
Podman (rootless) für alle Container.

*Begründung:*
- Kein Daemon erforderlich
- Rootless = erhöhte Sicherheit
- Docker-kompatibel (gleiche Images)
- Systemd Quadlet Integration
